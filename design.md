# 校園跑腿 APP 設計規劃

## 品牌定位
- **名稱**：校園跑腿（Campus Errand）
- **定位**：校園內學生互助跑腿媒合平台
- **色調**：活力橙（主色）搭配乾淨白底，傳遞快速、可信賴的感受

## 色彩系統

| Token | Light | Dark | 用途 |
|-------|-------|------|------|
| `primary` | `#FF6B35` | `#FF8C5A` | 主要按鈕、強調元素 |
| `background` | `#FFFFFF` | `#121212` | 頁面背景 |
| `surface` | `#F7F8FA` | `#1E1E1E` | 卡片、輸入框背景 |
| `foreground` | `#1A1A2E` | `#F0F0F0` | 主要文字 |
| `muted` | `#8A8FA3` | `#9BA1A6` | 次要文字、佔位符 |
| `border` | `#E8EAF0` | `#2A2A3A` | 分隔線、邊框 |
| `success` | `#22C55E` | `#4ADE80` | 完成狀態 |
| `warning` | `#F59E0B` | `#FBBF24` | 進行中狀態 |
| `error` | `#EF4444` | `#F87171` | 錯誤、取消狀態 |

## 頁面清單

### 底部導覽（Tab Bar）
1. **首頁**（探索任務）- 瀏覽所有可接的任務
2. **發單**（建立任務）- 發布新的跑腿任務
3. **我的訂單**（訂單管理）- 查看發出與接受的訂單
4. **個人**（個人中心）- 個人資料、評價、設定

### 詳細頁面
- **任務詳情頁**：查看完整任務資訊、接單操作
- **建立任務頁**：填寫任務表單（多步驟）
- **訂單詳情頁**：追蹤訂單狀態、聯絡對方
- **評價頁**：完成任務後互相評分

## 各頁面內容規劃

### 首頁（探索任務）
- 頂部搜尋欄 + 篩選按鈕（類別、距離、報酬）
- 任務分類標籤列（全部、餐飲、文件、購物、其他）
- 任務卡片列表（FlatList）
  - 任務標題、類別標籤
  - 發布地點、截止時間
  - 報酬金額（醒目顯示）
  - 發單人暱稱與評分
- 空狀態插圖（無任務時）

### 發單頁（建立任務）
- 分步驟表單：
  1. 任務基本資訊（標題、類別、描述）
  2. 地點資訊（取件地點、送達地點）
  3. 報酬與截止時間
- 底部「發布任務」確認按鈕
- 表單驗證與錯誤提示

### 我的訂單頁
- 分頁切換：「我發出的」/ 「我接的」
- 訂單卡片（含狀態標籤：待接單、進行中、已完成、已取消）
- 點擊進入訂單詳情頁

### 個人頁
- 頭像 + 暱稱 + 評分星級
- 統計數字（發單數、接單數、完成率）
- 功能列表：
  - 我的評價
  - 設定暱稱
  - 關於 APP

### 任務詳情頁
- 完整任務資訊展示
- 發單人資訊卡（暱稱、評分）
- 任務狀態（待接單顯示「接受任務」按鈕）
- 接單後顯示聯絡方式提示

### 訂單詳情頁
- 訂單狀態時間軸
- 任務資訊摘要
- 操作按鈕（根據角色與狀態顯示）：
  - 接單方：「確認完成」
  - 發單方：「確認收貨」、「取消訂單」

## 核心使用者流程

### 發單流程
首頁 → 點擊底部「發單」→ 填寫任務表單 → 確認發布 → 返回首頁（任務出現在列表）

### 接單流程
首頁 → 瀏覽任務列表 → 點擊任務卡片 → 查看任務詳情 → 點擊「接受任務」→ 進入我的訂單

### 完成訂單流程
我的訂單 → 點擊進行中訂單 → 接單方點「確認完成」→ 發單方點「確認收貨」→ 互相評價 → 訂單完成

## 設計原則
- **單手操作**：主要操作按鈕置於拇指可及範圍（底部 1/3）
- **資訊層次**：報酬金額最大最醒目，其次是任務標題
- **狀態清晰**：每個訂單狀態有獨特顏色標籤
- **空狀態友善**：無資料時顯示引導性插圖與文字
